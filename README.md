# AI Code Review Desk

A focused macOS desktop application for reviewing local Git changes generated by
AI coding tools (Claude Code, Codex, Cursor, etc.).

Not an IDE — a fast, lightweight review tool. Open a repo, scan the changed
files, read the diff, search across the codebase, preview & apply global
replacements, edit when needed, stage/unstage/discard with safe confirmations,
and ship.

## Tech stack

- [Tauri 2](https://tauri.app) — Rust core + native macOS window
- [React 19](https://react.dev) + TypeScript + [Vite](https://vitejs.dev)
- Linear-inspired CSS (Inter + JetBrains Mono via Google Fonts), no UI framework
- Custom unified-diff renderer (side-by-side / inline / edit) with built-in syntax tokens
- [Zustand](https://zustand-demo.pmnd.rs) — minimal state
- Rust crates: `regex`, `walkdir`, `ignore` (gitignore-aware traversal)

## Prerequisites

- **macOS** (target platform for the MVP)
- [**Bun**](https://bun.com) ≥ 1.3
- **Rust toolchain** via [rustup](https://rustup.rs):
  ```sh
  curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
  ```
- **Xcode Command Line Tools**
- **`git`** in `$PATH`
- Optional: **`code` shell command** for "Open in VS Code"

## Getting started

```sh
bun install
bun run tauri:dev
```

The first launch compiles Rust (a few minutes). Frontend hot-reload is instant
afterward.

Release bundle:

```sh
bun run tauri:build
```

## Keyboard shortcuts

| Shortcut         | Action                                |
| ---------------- | ------------------------------------- |
| ⌘ O              | Open repository                       |
| ⌘ P / ⌘ K        | Command palette                       |
| ⌘ R              | Refresh Git status                    |
| ⌘ ⇧ F            | Open global search                    |
| ⌘ ⇧ H            | Open replace                          |
| ⌘ ↩              | Stage / unstage current file          |
| ⌘ E              | Edit current file                     |
| ⌘ S              | Save edit                             |
| ⌘ ⌫              | Discard changes to current file       |
| ⌥ ↓ / ⌥ ↑        | Next / previous changed file          |
| Esc              | Close panel / overlay                 |

## Themes

Three themes ship out of the box: **Dark** (default), **Light**, **Midnight**.
Switch via the command palette → *Switch theme: Dark/Light/Midnight*.

## Safety model

- **Discard** always requires explicit confirmation via modal.
- **Replace** never writes to disk without an explicit preview + apply step.
  Each occurrence is individually selectable.
- The app never commits, pushes, or creates PRs.
- The app never deletes user files outside the working tree.

## AI features (preview only)

The right column flips to an AI panel when you click an AI Assist action. Tabs:

- Generate commit message
- Draft PR description
- Summarize this diff
- Review risk

These currently render preview placeholders — the `AiReviewService` interface
at `src/features/ai/ai.types.ts` is ready to plug into a real provider.

## Notes

- The diff renderer parses `git diff` unified output into hunks and renders
  side-by-side, inline, or an edit-mode textarea (writes via `git
  write_working_file`).
- Search & replace are gitignore-aware and skip files larger than 2 MiB and
  binary files.
- Git operations shell out to the `git` binary rather than libgit2 — small
  binary, identical behavior to the user's CLI.
- "Generated by" pill is detected heuristically from the last ten commit
  messages (looks for `Claude Code`, `Co-Authored-By: Claude`, etc.).
