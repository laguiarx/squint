# Releasing Dispatch

Dispatch uses Tauri updater artifacts hosted on GitHub Releases. The app checks:

```text
https://github.com/laguiarx/dispatch/releases/latest/download/latest.json
```

## One-time GitHub setup

Add these repository secrets in GitHub:

```text
TAURI_SIGNING_PRIVATE_KEY
TAURI_SIGNING_PRIVATE_KEY_PASSWORD
```

`TAURI_SIGNING_PRIVATE_KEY` must contain the private key generated by:

```sh
bun tauri signer generate --ci --write-keys ~/.tauri/dispatch-updater.key
```

For the key generated during this setup, the private key file is:

```text
/Users/lucasaguiar/.tauri/dispatch-updater.key
```

The password secret can be left empty if the key was generated without a
password. Keep the private key backed up; losing it means already-installed apps
cannot verify future updates.

## Cut a release

1. Update the app version in `package.json`, `src-tauri/Cargo.toml`, and
   `src-tauri/tauri.conf.json`.
2. Commit the version bump.
3. Create and push a tag:

```sh
git tag v0.2.4
git push origin v0.2.4
```

GitHub Actions will build macOS Apple Silicon and Intel bundles and create a
draft release. The action also uploads updater assets and `latest.json`.

## Publish

Before publishing the draft release, confirm the assets include:

```text
latest.json
*.sig
*.app.tar.gz
*.dmg
```

Once the draft is published, installed apps can discover the update from
`latest.json`. Draft releases are not enough for end-user updates.

## Manual release run

The same workflow can be run from GitHub Actions with a `v*` tag input, for
example:

```text
v0.2.4
```

Prefer tag pushes for normal releases because the git history then clearly shows
which commit produced each shipped build.
